# gitwrap

gitwrap is a small command-line wrapper around common Git operations. It adds safer defaults for destructive commands and returns machine-readable YAML output that can be used in scripts or automation workflows.

## Features

* YAML output for Git status information
* Confirmation prompt before deleting untracked files
* Global `--dry-run` support
* Global `--favourite-pokemon` metadata field (default: `Pikachu`)
* No external Python dependencies

## Project structure

```
gitwrap/
├── gitwrap.py       # CLI entry point and command routing
├── git_ops.py       # Git subprocess integration and parsing
├── yaml_output.py   # YAML serialization
├── test_gitwrap.py  # Unit tests
└── .gitignore
```

## Requirements

* Python 3.7+
* Git installed and available in your PATH

This project uses only the Python standard library.

## Usage

Run commands from inside a Git repository:

```bash
python gitwrap.py <command> [options]
```

### Global options

* `--dry-run` - show what would happen without making changes
* `--favourite-pokemon <name>` - include a metadata field in the YAML output

The global options can be placed before or after the subcommand.

## Commands

### status

Display the current repository status as YAML.

```bash
python gitwrap.py status
python gitwrap.py status --favourite-pokemon Gengar
python gitwrap.py --dry-run status
```

Example output:

```yaml
action: status
branch: main
staged_files:
  - src/app.js
unstaged_files:
  - README.md
untracked_files:
  - temp.txt
favourite_pokemon: Gengar
```

Only non-empty file categories are included in the output.

### clean

Preview or remove untracked files using `git clean -fd`.

#### Dry-run preview

```bash
python gitwrap.py clean --dry-run
python gitwrap.py --dry-run clean
```

Example output:

```yaml
dry_run: true
action: clean
favourite_pokemon: Pikachu
files:
  - notes.md
  - temp.txt
```

#### Interactive clean

```bash
python gitwrap.py clean
```

The tool will ask for confirmation before deleting files:

```text
This will delete 2 untracked files. Continue? [y/N]:
```

#### Skip confirmation

```bash
python gitwrap.py clean --yes
```

## Validation and error handling

Before running a command, gitwrap verifies that:

* Git is installed and available in PATH
* The current directory is inside a Git repository

If either check fails, the tool prints a clear error message and exits.

## Testing

Run the unit tests with:

```bash
python test_gitwrap.py
```

The current tests validate the YAML serialization logic.

## Implementation notes

* `status` uses `git status --porcelain=v2 --branch` for deterministic parsing.
* `clean` uses `git clean -fdn` to calculate the files that would be removed before executing the actual clean.
* YAML output is generated by a small custom serializer in `yaml_output.py`, avoiding external dependencies.
